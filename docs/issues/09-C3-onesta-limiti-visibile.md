# [C3] Rendere più visibili i limiti dei dati

**Blocco:** C — Allineato al principio d'oro
**Etichette:** valore: medio · costo: basso · area: ux+onestà

## Cosa fare
Mostrare meglio dove i dati sono approssimati o contesi. Non è un difetto da
nascondere: è onestà verso chi insegna.

## Passi
1. Esponi `BORDERPRECISION` dei confini quando disponibile (la fonte è approssimata,
   a scala continentale).
2. Segnala in UI dove la storia è contesa (date/confini con versioni in conflitto).
3. Mantieni e rafforza il conteggio "quanti eventi risolti" già presente nel pannello.

## Fatto quando
L'utente capisce a colpo d'occhio quanto un confine o una data sono affidabili.
