# Come trasformare questi bigliettini in issue su GitHub

Ogni file qui dentro è un "lavoro da fare", pronto da incollare su GitHub.
Li ho numerati nell'ordine in cui conviene farli: parti dal 01 e scendi.

## I passi (dal browser, niente terminale)

1. Vai sul tuo repo su GitHub.
2. In alto clicca la scheda **Issues**.
3. Clicca il bottone verde **New issue**.
4. Apri uno di questi file. La **prima riga** (quella con il `#`) è il **titolo**:
   copiala nel campo *Title* di GitHub (senza il `#`).
5. Copia **tutto il resto** del file e incollalo nel riquadro grande (*Description*).
6. Clicca **Submit new issue**. Fatto: il bigliettino è diventato una issue.
7. Ripeti per ogni file.

Quando finisci un lavoro, su GitHub apri la sua issue e clicca **Close issue**:
così vedi sempre cosa manca.

## Scorciatoia con Claude Code

Una volta nel Codespace, puoi anche dire a Claude Code:
"Leggi i file nella cartella issue-github e creali come issue su GitHub."
Lui può farlo da solo con la GitHub CLI (`gh`), che in Codespaces è già pronta.

## Regola d'oro (vale per ogni lavoro)

L'IA traduce e spiega; coordinate, confini e date vengono solo da
historical-basemaps e Wikidata, mai inventati. Tieni sempre visibile
l'avviso "verifica prima dell'uso in classe".
